# AGENTS.md — aisearcharab.com

هذا الملف هو المرجع التشغيلي لأي وكيل برمجي أو مساعد ذكاء اصطناعي يعمل داخل المستودع.

## 1. المهمة

ابنِ منصة عربية Static-First للصحافة الاستقصائية الرقمية، OSINT، التحقق، تحليل البيانات، والأدوات البحثية. المرجع الأعلى هو:

- `docs/EXECUTIVE_BLUEPRINT.md`
- السياسات الأمنية والتحريرية داخل المستودع.
- اختبارات المشروع ومعايير القبول المحددة في كل مهمة.

## 2. قواعد القرار

1. افصل دائماً بين الحقيقة، الادعاء، التقدير، والاستنتاج.
2. لا تخترع مصدراً، رقماً، اقتباساً، كياناً، أو نتيجة فحص.
3. لا تعتبر مخرجات نموذج لغوي دليلاً.
4. لا تنشر بيانات شخصية أو حساسة داخل مستودع عام.
5. لا تضف اعتماداً برمجياً جديداً إذا أمكن تنفيذ المهمة بأدوات المشروع الحالية.
6. فضّل البناء الثابت والحساب المسبق على الوظائف الديناميكية.
7. لا تضف JavaScript بلا ضرورة وظيفية واختبار أثره.
8. لا تستخدم أسراراً أو مفاتيح داخل الكود أو المحتوى أو السجل.
9. لا تغيّر البنية أو نماذج البيانات دون توثيق القرار.
10. لا تعلن نجاح المهمة قبل تشغيل الاختبارات ذات الصلة.

## 3. سير العمل الإلزامي

### قبل التنفيذ

- اقرأ الملفات ذات الصلة.
- افهم معيار القبول.
- حدّد أثر التغيير على المحتوى، البيانات، الأمان، الأداء، والوصول.
- تحقق من عدم وجود تنفيذ قائم يمكن إعادة استخدامه.

### أثناء التنفيذ

- نفّذ أصغر تغيير متماسك يحقق الهدف.
- حافظ على الفصل بين المحتوى والمنطق والعرض.
- أضف أنواعاً أو Schemas واضحة للبيانات المنظمة.
- عالج حالات الخطأ والمدخلات غير الصالحة.
- اكتب تعليقات فقط عندما تشرح قراراً غير بديهي.

### بعد التنفيذ

- شغّل اختبارات البناء والتحقق.
- افحص الروابط والـSchema عند تعديل المحتوى.
- افحص الوصول والأداء عند تعديل الواجهة.
- لخّص الملفات المتغيرة، سبب التغيير، الاختبارات، والمخاطر المتبقية.

## 5. معايير المحتوى

كل مادة استقصائية جديدة يجب أن تحدد:

- سؤال التحقيق.
- الفرضية.
- المنهجية.
- المصادر.
- الادعاءات الجوهرية.
- درجة الثقة.
- القيود.
- تاريخ آخر تحقق.
- المراجع أو المراجعين.
- حالة حق الرد، عند انطباقها.

لا تضع رابطاً في النص دون تسجيله ضمن نموذج المصادر عندما يكون دليلاً جوهرياً.

## 6. معايير البيانات

- المعرفات ثابتة، صغيرة الأحرف، ومفصولة بشرطات.
- التواريخ ISO 8601.
- الترميز UTF-8.
- الحقول المطلوبة لا تُملأ بقيم وهمية.
- القيمة غير المعروفة تكون `null` أو تُحذف وفق الـSchema؛ لا تستخدم "N/A" عشوائياً.
- احتفظ بالمصدر الأصلي لكل تحويل بيانات.
- لا تعدّل Dataset خاماً يدوياً؛ أنشئ طبقة مشتقة قابلة لإعادة البناء.

## 7. معايير Python والوكلاء

- Python 3.12 أو الإصدار المعتمد في المشروع.
- استخدم type hints.
- افصل I/O عن منطق التحويل والتحليل.
- ضع حدوداً للمهلة، إعادة المحاولة، وحجم المدخلات.
- سجّل الأخطاء دون تسريب بيانات حساسة.
- اجعل التشغيل idempotent كلما أمكن.
- كل Agent ينتج تقرير تشغيل machine-readable.
- لا تنفذ scraping مخالفاً لشروط الاستخدام أو الحواجز التقنية.
- لا تتجاوز CAPTCHA أو المصادقة أو التحكم في الوصول.

## 8. معايير الواجهة

- العربية وRTL افتراضياً.
- HTML دلالي قبل مكونات العرض.
- WCAG 2.2 AA هدف أدنى.
- تنقل بلوحة المفاتيح.
- دعم قارئات الشاشة وتقليل الحركة.
- لا تعتمد على اللون وحده لشرح حالة أو نتيجة.
- لا تجعل Glassmorphism يخفض التباين أو قابلية القراءة.
- الرسوم والشبكات تعرض فقط عندما تضيف قيمة تحليلية.

## 9. معايير الأداء

- لا تضف مكتبة Frontend كاملة لمكون بسيط.
- استخدم Hugo Pipes للأصول عند توفره.
- حسّن الصور وأحجامها قبل النشر.
- حمّل السكربتات غير الحرجة بتأخير مناسب.
- راقب حجم الحزم والصفحات.
- لا تقدّم ادعاءات أداء من اختبار محلي منفرد.

## 10. معايير الأمان

- أقل صلاحيات لـGitHub Actions.
- لا تستخدم `pull_request_target` مع كود غير موثوق دون عزل صارم.
- لا تطبع Secrets في Logs.
- ثبّت الاعتماديات وراجع تحديثاتها.
- عالج XSS في أي محتوى أو HTML مولد.
- افترض أن كل مدخل خارجي غير موثوق.
- لا تحفظ مواد تحقيق حساسة في المستودع العام.

## 11. Git وPull Requests

- Commit واحد لكل تغيير منطقي متماسك عندما يكون ذلك عملياً.
- رسائل Commit بصيغة واضحة مثل:
  - `feat: add source provenance schema`
  - `fix: reject invalid claim confidence values`
  - `docs: document correction workflow`
  - `chore: pin workflow dependencies`
- لا تعِد كتابة تاريخ الفرع الرئيسي.
- لا تدمج تغييراً يفشل في الاختبارات.

## 12. تعريف الإنجاز

المهمة مكتملة فقط عندما:

- تحقق معيار القبول.
- ينجح البناء.
- تنجح الاختبارات ذات الصلة.
- تكون البيانات والمصادر قابلة للتتبع.
- لا توجد أسرار أو بيانات حساسة.
- توثَّق القرارات غير البديهية.
- تُذكر القيود والمخاطر المتبقية بوضوح.
